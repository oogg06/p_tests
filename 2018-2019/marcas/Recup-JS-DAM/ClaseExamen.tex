\modulo{Lenguajes de marcas y sistemas de gestión de información}


\pregunta{Una popular franquicia de cafeterías desea ofrecer a sus clientes una aplicacion web para poder pedir su café de camino al trabajo.
Dados los siguientes requisitos y el HTML que se muestra, crear el fichero Javascript que consigue los resultados deseados}{10}

\begin{itemize}
\item{El usuario debe introducir su nombre  y los requisitos para preparar su café. Al final, en un {\tt div} con el {\tt id mensajes} aparecerá el texto ``Pepe, tu café costará 2.40 euros''}
\item{Hay tres tipos de tamaño: corto (1.10 euros), largo (1.40) y extra-largo (1.90).}
\item{Se puede tener con leche (0.35 euros más) o sin leche (0 euros más).}
\item{Se puede pedir un extra de azúcar (0.15 euros más) y con un vaso personalizado (0.80 euros más)}
\end{itemize}

Se debe tener en cuenta lo siguiente:
\begin{itemize}
\item{Se ha descubierto que determinadas combinaciones no parecer crear el tipo de sabor que caracteriza a la marca por lo que
no se permite que el usuario elija un café extra-largo sin leche pero con extra de azúcar. Si ocurre esto se debe desmarcar todo y escribir un mensaje de error en el {\tt div} cuyo {\tt id} es {\tt errores}. Dicho mensaje debe llevar un fondo rojo creado desde Javascript}
\item{Está permitido modificar el HTML.}
\item{Al cargar la página no hay ningún elemento marcado ni ningún texto escrito.}
\end{itemize}
